AI Declaration

การพัฒนาโปรเจกต์นี้ ได้ใช้ (AI) 
เป็นตัวช่วยสนับสนุนกระบวนการทำงานในบางขั้นตอน โดยมีขอบเขตการใช้งานดังนี้

1. Mock Data และ Unit Testing 

สร้าง mock data สำหรับจำลอง Incident, User, และ Chat message
ออกแบบโครงสร้างข้อมูลให้ทำงานนกับ domain model จริงได้
สร้างตัวอย่าง unit test สำหรับตรวจสอบ component และ business logic บางส่วน
แนะนำแนวทางtest edge case

mock data และ test case ได้รับการปรับแก้ให้ตรงกับ requirement จริงของระบบ และผ่านการตรวจสอบโดยทีมพัฒนาเอง

2. การแก้ไขปัญหาระบบ Chat

จัดการstateของchatที่ผูกกับ incidentId message ไม่อัปเดตตามสถานะล่าสุด
การจัดโครงสร้างcomponentและ lifecycle hook การจัดการ validation ของไฟล์แนบ

นำมาปรับแก้ และทดสอบด้วยตนเองก่อนใช้งานจริง

3. การทดสอบระบบ (UAT Testing Support)

สร้าง checklist สำหรับ UAT ตามแต่ละ role Admin,Driver,Passenger
จำลองscenarioการใช้งาน เช่น การ submit incident → เปลี่ยนสถานะ → เปิดแชท
วิเคราะห์ความครบถ้วนของ business flow

ตรวจสอบร่วมกับการทดสอบจริงเพื่อยืนยันความถูกต้อง

4. การออกแบบและพัฒนา Prototype FormและDashboard

เสนอแนวทางUXสำหรับฟอร์มรายงานเหตุการณ์
ออกแบบโครงสร้าง Dashboard แสดงสถานะ Incident
สร้างตัวอย่างcode prototype สำหรับlayoutและcomponentบางส่วน
ปรับปรุงโครงสร้างโค้ดให้เป็นระเบียบมากขึ้น

refactor และปรับแต่งให้เหมือนกับ architecture ของโปรเจกต์ก่อนใช้งานจริง


AI ถูกใช้ในฐานะผู้ช่วย ไม่ได้ถูกใช้ในการสร้างระบบแบบอัตโนมัติ
ทุกส่วนของโค้ด เอกสาร และกระบวนการทดสอบ ได้รับการตรวจสอบ แก้ไข และยืนยันความถูกต้องโดยทีมพัฒนา

ทีมงานรับผิดชอบต่อความถูกต้องของระบบทั้งหมดด้วยตนเอง
